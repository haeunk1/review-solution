# PRD (Product Requirements Document)
# Review AI Solution

**버전:** 1.0
**작성일:** 2026-03-17
**상태:** 진행 중

---

## 1. 제품 개요

### 문제 정의
피부과·성형외과 등 의료 미용 클리닉은 온라인 리뷰가 매출에 직접적인 영향을 미친다. 그러나 네이버 지도, 구글 맵, 강남언니 등 다수 플랫폼에 흩어진 리뷰를 수동으로 모니터링하고 답글을 다는 것은 인력 낭비이며 누락이 발생하기 쉽다.

### 솔루션
리뷰 자동 수집 → AI 분석 → AI 답글 생성 → 웹 대시보드에서 컨펌 후 게시하는 전 과정을 자동화한 SaaS 서비스.

### 타겟 고객
- 1차: 피부과, 성형외과
- 2차: 치과, 한의원, 일반 병원 등 의료기관 전반

## 2. 핵심 기능 요구사항

### 2.1 리뷰 수집 (최우선)
| 항목 | 내용 |
|------|------|
| 지원 플랫폼 | 네이버 지도, 구글 맵, 강남언니/힐링페이퍼 |
| 수집 방식 | Playwright 기반 웹 크롤링 |
| 수집 주기 | 클리닉별 설정 가능 (1h / 3h / 6h / 24h) |
| 중복 방지 | platform + platform_review_id 기반 중복 체크 |
| 수동 트리거 | 대시보드에서 즉시 수집 버튼 제공 |

### 2.2 AI 분석
| 항목 | 내용 |
|------|------|
| 감성 분석 | 긍정 / 부정 / 중립 분류 |
| 키워드 추출 | 자주 등장하는 단어/주제 클러스터링 |
| 평점 통계 | 플랫폼별, 기간별 평점 트렌드 |
| 경쟁사 모니터링 | 근처 경쟁 클리닉 리뷰 수집 및 비교 |

### 2.3 AI 답글 생성
| 항목 | 내용 |
|------|------|
| 답글 스타일 | formal(정중), friendly(친근), positive(긍정적) |
| 자동 스타일 선택 | 부정 리뷰 → formal, 일반 → friendly, 긍정 → positive |
| 수동 스타일 선택 | 사용자가 스타일 직접 선택 후 재생성 가능 |
| AI 모델 | OpenAI GPT-4o |

### 2.4 답글 컨펌 & 게시
| 항목 | 내용 |
|------|------|
| 승인 방식 | 웹 대시보드에서 확인 후 승인 버튼 클릭 |
| 수정 가능 | 승인 전 답글 직접 수정 가능 |
| 게시 방식 | 승인 후 Playwright로 해당 플랫폼에 자동 게시 |
| 상태 관리 | pending → approved → posted |

### 2.5 멀티테넌시 (SaaS)
| 항목 | 내용 |
|------|------|
| 테넌트 구분 | 클리닉별 완전한 데이터 분리 |
| 사용자 권한 | admin(전체 관리) / staff(조회·승인만) |
| 인증 | JWT 기반 로그인 |

## 3. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 보안 | 테넌트 간 데이터 격리, JWT 인증 필수 |
| 확장성 | 신규 플랫폼 스크래퍼 추가 용이한 구조 (BaseScraper) |
| 안정성 | 크롤링 실패 시 재시도 없이 로그 기록 후 다음 주기에 재수집 |
| 플랫폼 변경 대응 | CSS 셀렉터 변경 시 코드 수정만으로 대응 가능한 구조 |

## 4. 기술 스택

| 영역 | 기술 |
|------|------|
| Backend | FastAPI, Python 3.11+ |
| Database | PostgreSQL + SQLAlchemy + Alembic |
| Scraping | Playwright (Chromium) |
| Scheduling | APScheduler |
| AI | OpenAI GPT-4o API |
| Auth | JWT (python-jose, passlib) |
| Frontend | Next.js + TailwindCSS |
| 배포 (예정) | Docker + AWS / GCP |
